# Kubernetes cluster (kubeadm) on OCI Oracle Linux 9

Ansible automation that builds a two-node Kubernetes cluster on the Oracle Linux 9
VMs provisioned by the Terraform stack in [`../terraform`](../terraform):

| Role in cluster | Terraform VM key | Inventory group     |
| --------------- | ---------------- | ------------------- |
| Control plane   | `control_plane`  | `k8s_control_plane` |
| Worker node     | `worker_node`    | `k8s_workers`       |

The playbook installs a **containerd** runtime, **Kubernetes v1.36** (via the
official `pkgs.k8s.io` repo) with **kubeadm**, and the **Calico** CNI. All
versions are pinned in [`inventory/group_vars/all.yml`](inventory/group_vars/all.yml)
so upgrades are a one-line change.

## Prerequisites

- The Terraform stack in `../terraform` has been applied and both VMs are running.
- Ansible installed on your workstation (the control node):

  ```bash
  python3 -m pip install --user ansible
  ```

- The SSH private key generated by Terraform at
  `../terraform/generated/sd-linux` (referenced by [`ansible.cfg`](ansible.cfg)).

## 1. Build the inventory

From `../terraform`, read the VM addresses:

```bash
cd ../terraform
terraform output instance_public_ips     # -> ansible_host (SSH)
terraform output instance_private_ips    # -> node_ip (in-cluster address)
```

Copy the example inventory and fill in the IPs:

```bash
cd ../ansible
cp inventory/hosts.yml.example inventory/hosts.yml
$EDITOR inventory/hosts.yml
```

Put the `control_plane` VM under `k8s_control_plane` and the `worker_node` VM
under `k8s_workers`. `ansible_host` is the public IP (SSH); `node_ip` is the
private IP the node advertises to the cluster.

## 2. Verify connectivity

```bash
ansible all -m ping
```

## 3. Deploy the cluster

```bash
ansible-playbook site.yml
```

This runs three stages: prepare every node, `kubeadm init` on the control plane
plus Calico, then join the worker node.

Useful subsets:

```bash
ansible-playbook site.yml --tags common              # node prep only
ansible-playbook site.yml --limit k8s_control_plane  # control plane only
ansible-playbook site.yml --limit k8s_workers --tags worker
```

## 4. Use the cluster

SSH to the control plane and run `kubectl` as the `opc` user (kubeconfig is
installed to `~/.ssh`/`~/.kube/config` during the run):

```bash
ssh -i ../terraform/generated/sd-linux opc@<control_plane_public_ip>
kubectl get nodes -o wide
kubectl get pods -A
```

Both nodes should report `Ready` once Calico finishes rolling out.

## Layout

```
docker-k8s/ansible/
├── ansible.cfg                  # opc user, terraform SSH key, sudo
├── site.yml                     # 3 plays: common -> control plane -> worker
├── inventory/
│   ├── hosts.yml.example        # groups k8s_control_plane and k8s_workers
│   └── group_vars/all.yml       # pinned versions and CIDRs
└── roles/
    ├── k8s_common/              # runtime + packages + sysctl + firewall (all nodes)
    ├── k8s_control_plane/       # kubeadm init + kubeconfig + Calico
    └── k8s_worker/              # kubeadm join
```

## Notes

- The OCI security list already permits all traffic within the VCN
  (`10.0.0.0/16`), so in-cluster ports are additionally opened in each node's
  firewalld by the `k8s_common` role for defense in depth.
- The pod CIDR (`192.168.0.0/16`, Calico default) does not overlap the VCN, so
  there are no route conflicts.
- Calico uses VXLAN encapsulation because the OCI VCN does not run BGP and drops
  IPIP; VXLAN is plain UDP/4789 which the VCN forwards.
